|---------|------|
| 项目名称 | LEMS - Lab Equipment Management System |

## 1. 环境准备

### 1.1 软件清单

在部署 LEMS 系统之前，需要安装以下软件。请严格按照以下版本进行安装，版本不一致可能导致兼容性问题。

| 软件 | 版本 | 下载地址 | 说明 |
|------|------|---------|------|
| JDK | 1.8.0_361 (Temurin 8) | https://adoptium.net/temurin/releases/ | 选择 Windows x64 .msi 安装包 |
| Maven | 3.8.8+ | https://maven.apache.org/download.cgi | 选择 Binary zip archive |
| MySQL | 8.0.36 | https://dev.mysql.com/downloads/ | 选择 MySQL Installer for Windows |
| Nacos | 2.2.3 | https://github.com/alibaba/nacos/releases | 选择 nacos-server-2.2.3.zip |
| Redis | 5.0.14 (Windows) | https://github.com/tporadowski/redis/releases | 选择 Redis-x64-5.0.14.zip |
| Node.js | 18.20.0 LTS | https://nodejs.org/ | 选择 Windows Installer (.msi) |
| Nginx | 1.24.0 | https://nginx.org/en/download.html | 选择 nginx-1.24.0.zip (Windows) |

### 1.2 JDK 安装与配置

1. 双击下载的 `.msi` 安装包，按提示完成安装。建议安装路径为 `C:\Program Files\Eclipse Adoptium\`。
2. 配置环境变量：
   - 新建系统变量 `JAVA_HOME`，值为 JDK 安装路径（如 `C:\Program Files\Eclipse Adoptium\jdk8u361-b09`）。
   - 编辑系统变量 `Path`，新增 `%JAVA_HOME%\bin`。
3. 验证安装：打开 CMD 窗口，执行 `java -version`，应显示 `1.8.0_361`。

### 1.3 Maven 安装与配置

1. 下载 `apache-maven-3.8.8-bin.zip` 并解压到 `C:\apache-maven-3.8.8`。
2. 配置环境变量：
   - 新建系统变量 `MAVEN_HOME`，值为 `C:\apache-maven-3.8.8`。
   - 编辑系统变量 `Path`，新增 `%MAVEN_HOME%\bin`。
3. 验证安装：打开 CMD 窗口，执行 `mvn -version`，应显示 `Apache Maven 3.8.8`。
4. （可选）配置 Maven 镜像：编辑 `C:\apache-maven-3.8.8\conf\settings.xml`，在 `<mirrors>` 节点中添加阿里云镜像加速。

### 1.4 MySQL 安装与配置

1. 下载 MySQL Installer，选择 "Server only" 安装类型。
2. 安装过程中设置 root 密码为 `root`（与项目配置一致）。
3. 配置端口为 `3306`，字符集选择 `utf8mb4`。
4. 验证安装：打开 CMD，执行 `mysql -u root -p`，输入密码后进入 MySQL 命令行。
5. 创建数据库并导入初始数据：

```bash
mysql -u root -p --default-character-set=utf8mb4 < "D:\000\LEMS-main\sql\init.sql"
```

或者使用 MySQL Workbench 等图形工具打开 `sql/init.sql` 文件并执行。

### 1.5 Redis 安装

1. 下载 `Redis-x64-5.0.14.zip` 并解压到 `C:\Redis`。
2. 双击 `redis-server.exe` 启动 Redis 服务（默认端口 6379）。
3. 验证安装：新开 CMD 窗口，执行 `C:\Redis\redis-cli.exe ping`，应返回 `PONG`。

### 1.6 Nacos 安装

1. 下载 `nacos-server-2.2.3.zip` 并解压到 `C:\nacos`。
2. 单机模式启动（Windows）：

```cmd
cd C:\nacos\bin
startup.cmd -m standalone
```

3. 验证安装：浏览器访问 `http://localhost:8848/nacos`，使用默认账号 `nacos/nacos` 登录。

### 1.7 Node.js 安装

1. 双击 `.msi` 安装包，按提示完成安装。
2. 验证安装：打开 CMD，执行 `node -v`（应显示 `v18.20.0`）和 `npm -v`（应显示 `10.5.0`）。

## 2. 后端部署

### 2.1 获取源码

从 GitHub 克隆项目到本地：

```cmd
git clone https://github.com/LWJSTONE/LEMS.git
cd LEMS
```

### 2.2 编译打包

```cmd
mvn clean package -DskipTests
```

编译完成后，在各模块的 `target` 目录下生成 JAR 文件：

```
lab-gateway/target/lab-gateway-1.0.0.jar
lab-auth/target/lab-auth-1.0.0.jar
lab-user/target/lab-user-1.0.0.jar
lab-device/target/lab-device-1.0.0.jar
lab-borrow/target/lab-borrow-1.0.0.jar
lab-report/target/lab-report-1.0.0.jar
```

### 2.3 修改数据库配置（如有需要）

如果 MySQL 的 root 密码不是 `root`，需要修改各微服务的 `application.yml` 文件中的数据库连接配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/lab_device_mgt?useUnicode=true&characterEncoding=UTF-8&zeroDateTimeBehavior=convertToNull&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: root  # 修改为实际密码
```

需要修改的文件：
- `lab-auth/src/main/resources/application.yml`
- `lab-user/src/main/resources/application.yml`
- `lab-device/src/main/resources/application.yml`
- `lab-borrow/src/main/resources/application.yml`
- `lab-report/src/main/resources/application.yml`

### 2.4 启动后端服务

**启动顺序很重要**，请严格按照以下顺序启动：

**方式一：使用一键启动脚本（推荐）**

双击项目根目录下的 `start-lab.bat` 文件，脚本会自动编译并按顺序启动所有服务。

**方式二：手动启动**

分别打开 6 个 CMD 窗口，按以下顺序执行：

```cmd
# 1. 启动网关（端口 8080）
java -jar lab-gateway/target/lab-gateway-1.0.0.jar

# 2. 启动认证服务（端口 8081）
java -jar lab-auth/target/lab-auth-1.0.0.jar

# 3. 启动用户服务（端口 8082）
java -jar lab-user/target/lab-user-1.0.0.jar

# 4. 启动设备服务（端口 8083）
java -jar lab-device/target/lab-device-1.0.0.jar

# 5. 启动借用服务（端口 8084）
java -jar lab-borrow/target/lab-borrow-1.0.0.jar

# 6. 启动报表服务（端口 8085）
java -jar lab-report/target/lab-report-1.0.0.jar
```

### 2.5 验证后端启动

1. 打开 Nacos 控制台 `http://localhost:8848/nacos`，在"服务列表"页面查看所有 6 个服务是否已注册且状态为"UP"。
2. 服务列表应显示：`lab-gateway`、`lab-auth`、`lab-user`、`lab-device`、`lab-borrow`、`lab-report`。
3. 使用 Postman 测试登录接口：`POST http://localhost:8080/api/v1/auth/login`，请求体 `{"username":"admin","password":"admin123"}`，应返回 Token。

## 3. 前端部署

### 3.1 安装依赖

```cmd
cd lems-ui
npm install
```

### 3.2 开发模式运行

```cmd
npm run dev
```

浏览器访问 `http://localhost:3000`，可以正常使用系统（开发模式通过 Vite 代理转发 API 请求到 Gateway :8080）。

### 3.3 生产构建

```cmd
npm run build
```

构建完成后，`lems-ui/dist` 目录包含所有静态资源文件。

### 3.4 Nginx 部署

1. 下载并解压 Nginx 到 `C:\nginx`。
2. 将 `lems-ui/dist` 目录下的所有文件复制到 `C:\nginx\html\lab` 目录。
3. 替换 `C:\nginx\conf\nginx.conf` 为项目提供的 `nginx.conf` 配置文件（已包含 API 反向代理配置）。
4. 启动 Nginx：

```cmd
cd C:\nginx
start nginx
```

5. 验证部署：浏览器访问 `http://localhost`，应显示系统登录页面。

### 3.5 Nginx 配置说明

项目提供的 `nginx.conf` 核心配置如下：

```nginx
server {
    listen       80;
    server_name  localhost;

    # 前端静态资源
    location / {
        root   html/lab;
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理到 Gateway
    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

## 4. 完整启动流程（从零开始）

以下是全新 Windows 环境下部署 LEMS 系统的完整步骤清单：

1. ✅ 安装 JDK 1.8，配置 `JAVA_HOME` 环境变量
2. ✅ 安装 Maven 3.8+，配置 `MAVEN_HOME` 环境变量
3. ✅ 安装 MySQL 8.0，设置 root 密码，创建数据库并执行 `sql/init.sql`
4. ✅ 安装并启动 Redis（双击 `redis-server.exe`）
5. ✅ 安装 Nacos，执行 `startup.cmd -m standalone` 启动
6. ✅ 安装 Node.js 18
7. ✅ 克隆项目源码 `git clone https://github.com/LWJSTONE/LEMS.git`
8. ✅ 编译后端 `mvn clean package -DskipTests`
9. ✅ 按顺序启动 6 个微服务（或使用 `start-lab.bat`）
10. ✅ 前端构建 `cd lems-ui && npm install && npm run build`
11. ✅ 将 `dist` 复制到 Nginx 目录，配置 `nginx.conf`
12. ✅ 启动 Nginx，访问 `http://localhost`

## 6. 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| Nacos 中服务显示 DOWN | 服务启动失败 | 查看对应服务的控制台日志，检查数据库连接和 Redis 连接 |
| 登录返回 500 | 数据库未初始化 | 确认已执行 `sql/init.sql` 脚本 |
| 前端页面空白 | Nginx 配置路径错误 | 检查 `nginx.conf` 中的 `root html/lab` 路径是否正确 |
| API 请求 404 | Gateway 未启动或路由错误 | 确认 Gateway 服务已启动（端口 8080） |
| API 请求 503 | 下游服务未启动 | 检查 Nacos 控制台，确认目标服务已注册且状态为 UP |
| 借用申请报"操作频繁" | Redis 未启动 | 启动 Redis：`redis-server.exe` |
| 数据库连接失败 | 密码不匹配 | 检查 `application.yml` 中的数据库密码配置 |
| 编译失败 | Maven 依赖下载超时 | 配置 Maven 阿里云镜像，或检查网络连接 |

## 7. 停止服务

**停止 Nginx**：
```cmd
cd C:\nginx
nginx -s stop
```

**停止 Java 服务**：在每个 Java 服务的 CMD 窗口中按 `Ctrl + C`。

**停止 Redis**：在 Redis 的 CMD 窗口中按 `Ctrl + C`，或执行 `redis-cli shutdown`。

**停止 Nacos**：在 Nacos 的 CMD 窗口中按 `Ctrl + C`。
